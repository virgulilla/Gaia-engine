# UI-004 — Notifications

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines the notification system used to communicate important events to the player.

Notifications provide timely feedback without interrupting gameplay.

---

# Scope

This specification defines:

- notification types
- priorities
- display rules
- notification lifecycle
- notification history

It does not define:

- achievements
- objectives
- gameplay logic
- rendering

---

# Philosophy

Notifications should inform.

They should never overwhelm the player.

Only meaningful events deserve immediate attention.

---

# Responsibilities

The Notification System is responsible for:

- displaying notifications
- prioritizing events
- grouping similar events
- maintaining notification history

The Notification System never:

- modify gameplay
- execute simulation logic
- own gameplay state

---

# Notification Pipeline

```text
Simulation Event

↓

Gameplay Event

↓

Notification Request

↓

Priority Evaluation

↓

Display Queue

↓

HUD
```

---

# Notification Categories

```text
Notifications

├── Information
├── Discovery
├── Objective
├── Achievement
├── Warning
├── Critical
└── System
```

---

# Information

General simulation updates.

Examples:

- Season Changed
- Rain Started
- New Population Peak

---

# Discovery

Generated by the Discovery System.

Examples:

- New Species
- New Trait
- New Biome
- New Resource

Discovery notifications include an Encyclopedia shortcut.

---

# Objective

Examples:

- Objective Started
- Objective Updated
- Objective Completed

Objectives should display progress when appropriate.

---

# Achievement

Examples:

- Achievement Unlocked
- Milestone Reached

Achievements use distinctive visual styling.

---

# Warning

Examples:

- Species Near Extinction
- Food Shortage
- Drought
- Population Collapse

Warnings require player attention.

---

# Critical

Examples:

- Simulation Error
- Save Failure
- World Corruption

Critical notifications require acknowledgment.

---

# System

Examples:

- World Saved
- Auto Save Complete
- Settings Updated

System notifications are low priority.

---

# Notification Structure

Every notification contains:

```text
Notification

├── NotificationId
├── Category
├── Priority
├── Title
├── Message
├── Timestamp
├── Duration
└── Action
```

---

# Priority Levels

Supported priorities:

- Low
- Normal
- High
- Critical

Higher priority notifications appear first.

---

# Queue

Notifications are displayed through a queue.

The queue guarantees:

- deterministic order
- no duplicates
- configurable limits

---

# Grouping

Repeated notifications may be grouped.

Example:

Instead of:

- Plant Discovered
- Plant Discovered
- Plant Discovered

Display:

- 3 New Plant Discoveries

---

# Duration

Default durations:

Low

3 seconds

Normal

5 seconds

High

8 seconds

Critical

Persistent until dismissed

---

# History

Every displayed notification is stored temporarily.

Players may review recent notifications.

History size is configurable.

---

# Interaction

Notifications may support:

- Open Encyclopedia
- Focus Camera
- Open Objective
- Dismiss

Actions are optional.

---

# Accessibility

Notifications support:

- scalable text
- screen readers
- configurable duration
- reduced motion

---

# Performance

Notification rendering should:

- reuse UI components
- avoid allocations
- minimize layout rebuilds

---

# Determinism

Notification order is deterministic.

Presentation timing never affects simulation.

---

# Serialization

Displayed notifications are not serialized.

Notification history may optionally be preserved between sessions.

---

# Design Constraints

The Notification System must remain:

- lightweight
- accessible
- deterministic
- simulation independent
