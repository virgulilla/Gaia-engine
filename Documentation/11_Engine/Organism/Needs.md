# ORG-007 — Needs

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines the motivational state of an organism.

Needs represent biological and environmental pressures that drive behaviour.

Needs never make decisions.

---

# Scope

This specification defines:

- biological needs
- priority calculation
- urgency
- satisfaction
- need lifecycle

It does not define:

- AI
- pathfinding
- movement
- actions
- behaviour selection

---

# Philosophy

Needs answer one question:

"What does the organism require right now?"

Behaviour answers a different question:

"What should the organism do?"

---

# Responsibilities

Needs are responsible for:

- representing internal motivation
- tracking urgency
- exposing priorities
- notifying Behaviour Systems of changes

Needs never:

- choose actions
- evaluate strategies
- execute movement
- modify physiology

---

# Runtime Structure

```text
Needs

├── Survival
├── Maintenance
├── Reproduction
├── Social
└── Environmental
```

---

# Survival Needs

Highest priority needs.

Examples:

- Hunger
- Thirst
- Oxygen
- Body Temperature

Failure leads to death.

---

# Maintenance Needs

Support long-term survival.

Examples:

- Rest
- Recovery
- Grooming
- Healing

---

# Reproduction Needs

Represent reproductive motivation.

Examples:

- Mate Search
- Nest Building
- Incubation
- Parenting

---

# Social Needs

Only present in social organisms.

Examples:

- Group Cohesion
- Dominance
- Protection
- Cooperation

Organisms without social behaviour ignore this category.

---

# Environmental Needs

Generated by environmental conditions.

Examples:

- Seek Shade
- Find Shelter
- Escape Flood
- Avoid Fire
- Leave Toxic Area

---

# Need Structure

Every Need contains:

```text
Need

├── Type
├── Current Value
├── Target Value
├── Priority
├── Urgency
├── Decay Rate
└── Recovery Rate
```

---

# Priority

Priority determines relative importance.

Priority is dynamic.

It changes continuously during the simulation.

---

# Urgency

Urgency represents how critical a need has become.

Typical range:

0.0 → 1.0

Higher urgency increases behavioural influence.

---

# Satisfaction

Needs may be:

- Satisfied
- Active
- Critical

Critical needs always override lower priority needs.

---

# Inputs

Needs receive information from:

- Physiology
- Body Instance
- Climate
- Resources
- Simulation Events

---

# Outputs

Needs provide input to:

- Behaviour System
- Utility AI
- Statistics

Needs never trigger actions directly.

---

# Runtime Rules

Needs are recalculated every Simulation Tick.

Priority values are continuous.

Binary state transitions should be avoided.

---

# Serialization

All Need values must be serializable.

Loading a saved simulation must restore identical motivation states.

---

# Design Constraints

Needs must remain:

- deterministic
- continuous
- data-driven
- behaviour independent
