# ORG-004 — Body Schema

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines the anatomical structure generated by Morphogenesis.

The Body Schema represents the organism's physical blueprint.

It does not represent the current biological state.

---

# Scope

This specification defines:

- anatomical layout
- structural characteristics
- physical capabilities
- body topology

It does not define:

- health
- injuries
- behaviour
- current energy
- diseases

Those belong to runtime systems.

---

# Philosophy

The Body describes:

"What the organism is."

It does not describe:

"What is happening to the organism."

---

```text
Genome
```

---

# Body Schema

The Body Schema is immutable after birth.

It represents the organism's anatomy.

---

# Body Sections

Every Body Schema contains:

```text
Body

├── Skeleton
├── Muscles
├── Locomotion
├── Sensors
├── Covering
├── Internal Systems
└── Special Structures
```

---

# Skeleton

Defines:

- body proportions
- structural strength
- body size
- limb attachment points
- joint configuration

---

# Muscles

Defines:

- muscle density
- strength potential
- movement efficiency

---

# Locomotion

Defines movement capabilities.

Examples:

Walking

Running

Swimming

Flying (future)

Burrowing (future)

Climbing (future)

The schema defines capability.

Current movement belongs to Movement System.

---

# Sensors

Defines:

Vision

Smell

Hearing

Touch

Electroreception (future)

Other sensory systems may be added.

---

# Covering

Examples:

Fur

Scales

Skin

Feathers

Shell

Covering affects:

temperature regulation

camouflage

protection

---

# Internal Systems

Defines anatomical capabilities.

Examples:

Digestive System

Respiratory System

Circulatory System

Reproductive System

These are structures.

They are not currently active processes.

---

# Special Structures

Optional structures.

Examples:

Horns

Tusks

Spikes

Tail

Pouch

Antlers

Future versions may introduce new structures.

---

# Body Blueprint

The Body Schema is effectively a blueprint.

It never changes.

It is shared conceptually across the organism lifetime.

---

# Runtime Body

Runtime modifications belong to Body Instance.

Examples:

Broken leg

Missing eye

Scar

Parasite

Pregnancy

None of these modify Body Schema.

---

# Constraints

Body Schema must remain:

immutable

serializable

deterministic

platform independent
