# ART-001 — Style Guide

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines the artistic direction of Gaia Engine.

Every visual asset must follow this guide to ensure a coherent visual identity across the entire project.

---

# Scope

This specification defines:

- visual identity
- artistic principles
- color language
- shape language
- asset consistency

It does not define:

- rendering
- shaders
- UI implementation
- gameplay

---

# Philosophy

The player should immediately understand the world through its visuals.

Art must improve readability before realism.

---

# Artistic Vision

Gaia Engine uses a stylized low-poly aesthetic.

The visual style emphasizes:

- readability
- simplicity
- color harmony
- procedural generation

Photorealism is not a goal.

---

# Visual Pillars

The art direction is based on five pillars.

## Readability

Important gameplay information must be recognizable at a glance.

---

## Simplicity

Avoid unnecessary geometric complexity.

Silhouettes are more important than details.

---

## Consistency

Every asset follows the same artistic language.

No asset should appear to belong to a different game.

---

## Procedural Compatibility

Assets must support procedural generation.

Every visual element should be modular.

---

## Performance

All assets must be suitable for mobile hardware.

Optimization is mandatory.

---

# Shape Language

Organic objects:

- rounded
- soft
- asymmetric

Natural structures:

- irregular
- layered
- weathered

Artificial structures (future):

- geometric
- symmetrical
- ordered

---

# Color Palette

Colors should be:

- vibrant
- natural
- slightly saturated

Avoid:

- pure black
- pure white
- excessive contrast

Environmental colors should communicate biome identity.

---

# Lighting

Preferred lighting:

- soft shadows
- warm sunlight
- strong ambient lighting

Lighting should improve readability.

---

# Materials

Materials should be stylized.

Examples:

- wood
- stone
- grass
- fur
- water

Avoid excessive texture detail.

Color variation is preferred over texture complexity.

---

# Animation Style

Animations should be:

- readable
- expressive
- lightweight

Exaggerated motion is encouraged.

---

# Visual Feedback

Player interactions should always produce visual feedback.

Examples:

- movement
- discovery
- resource collection
- weather changes

---

# Accessibility

Visual information must never rely exclusively on color.

Shape, animation and icons should reinforce communication.

---

# Mobile Constraints

Assets should prioritize:

- low polygon count
- efficient materials
- minimal draw calls
- texture reuse

---

# Design Constraints

The Style Guide must remain:

- renderer independent
- platform independent
- consistent
- scalable
