# WRLD-007 — Resources

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines every consumable and exploitable resource available in the World.

Resources are required for organism survival, ecosystem stability and long-term evolution.

---

# Scope

This specification defines:

- renewable resources
- non-renewable resources
- regeneration
- depletion
- distribution
- resource ownership

It does not define:

- organism behaviour
- resource gathering
- rendering
- gameplay economy

---

# Philosophy

Resources exist independently from organisms.

Organisms consume resources.

Resources never react to organisms directly.

Simulation Systems manage regeneration and depletion.

---

# Responsibilities

The Resource module is responsible for:

- resource availability
- regeneration
- depletion
- distribution
- environmental modifiers

The Resource module never:

- decides consumption
- controls organism behaviour
- generates terrain
- simulates weather

---

# Resource Structure

```text
Resources

├── Renewable
├── Non-Renewable
├── Biological
├── Geological
└── Environmental
```

---

# Renewable Resources

Resources that naturally recover.

Examples:

- Vegetation
- Fresh Water
- Fruits
- Seeds
- Fish Population

Each renewable resource defines:

- Maximum Capacity
- Current Amount
- Regeneration Rate

---

# Non-Renewable Resources

Resources that do not regenerate during normal gameplay.

Examples:

- Minerals
- Fossils
- Rare Crystals

---

# Biological Resources

Produced by living organisms.

Examples:

- Leaves
- Wood
- Fruits
- Meat
- Carcasses
- Eggs

Availability depends on ecosystem health.

---

# Geological Resources

Produced by Terrain.

Examples:

- Stone
- Clay
- Sand
- Metal Ores

Distribution depends on geology.

---

# Environmental Resources

Resources generated by environmental conditions.

Examples:

- Sunlight
- Wind
- Rainwater
- Geothermal Heat

These resources are not stored.

They are calculated from World conditions.

---

# Resource Properties

Every resource defines:

```text
Resource

├── ResourceId
├── Type
├── Current Amount
├── Maximum Capacity
├── Regeneration Rate
├── Quality
└── Availability
```

---

# Availability

Availability is continuous.

Range:

0.0 → 1.0

Availability influences:

- organism survival
- reproduction
- migration
- biodiversity

---

# Quality

Quality affects usefulness.

Examples:

- Fresh Water
- Polluted Water
- Rich Soil
- Poor Soil

Quality changes over time.

---

# Regeneration

Renewable resources regenerate according to:

- climate
- terrain
- biome
- season

Regeneration is calculated by dedicated systems.

---

# Depletion

Resources decrease when consumed.

Consumption never destroys the resource definition.

Only its current amount changes.

---

# Resource Distribution

Resources are stored per Chunk.

Each Chunk maintains its own resource state.

Neighbouring Chunks may exchange renewable resources through environmental systems.

---

# Climate Interaction

Climate influences:

- vegetation growth
- evaporation
- soil moisture
- food production

---

# Terrain Interaction

Terrain influences:

- fertility
- mineral availability
- drainage
- groundwater access

---

# Organism Interaction

Organisms may:

- consume resources
- compete for resources
- transport resources

Resource Systems never execute these actions.

---

# Determinism

Resource generation and regeneration must be deterministic.

Given identical simulation history, resource availability must always be identical.

---

# Serialization

Resource state must support complete serialization.

Saving and loading restores identical resource quantities.

---

# Design Constraints

Resources must remain:

- deterministic
- data-driven
- scalable
- renderer independent
- configurable
