# GAME-003 — Encyclopedia

**Version:** 1.0.0

**Status:** Approved

**Owner:** Gaia Engine Team

**Last Updated:** 2026-06-27

---

# Purpose

Defines the Encyclopedia, the permanent knowledge repository available to the player.

The Encyclopedia records discoveries generated by the simulation and presents them as structured information.

---

# Scope

This specification defines:

- encyclopedia entries
- categories
- unlock conditions
- progression
- historical records

It does not define:

- UI presentation
- tutorials
- gameplay objectives
- simulation logic

---

# Philosophy

The Encyclopedia never invents information.

Everything stored inside it must originate from player discoveries.

Knowledge is earned through observation.

---

# Responsibilities

The Encyclopedia is responsible for:

- storing discovered information
- organizing entries
- tracking completion
- exposing searchable knowledge

The Encyclopedia never:

- modify simulation data
- generate discoveries
- execute gameplay logic
- unlock content automatically

---

# Encyclopedia Structure

```text
Encyclopedia

├── Species
├── Traits
├── Biomes
├── Resources
├── Climate
├── Behaviours
├── Evolution
└── World History
```

---

# Species Entries

Every discovered species receives an entry.

Each entry may contain:

- Name
- Scientific Identifier
- Classification
- Habitat
- Diet
- Lifespan
- Population Statistics
- Extinction Status

Additional information becomes available through continued observation.

---

# Trait Entries

Stores biological characteristics.

Examples:

- Thick Fur
- Venom
- Camouflage
- Long Neck
- Large Brain
- Fast Metabolism

Traits may reference related species.

---

# Biome Entries

Contains information about:

- Climate
- Resources
- Typical Species
- Environmental Conditions

---

# Resource Entries

Contains information about:

- Resource Type
- Availability
- Regeneration
- Uses
- Distribution

---

# Climate Entries

Stores discovered environmental phenomena.

Examples:

- Blizzard
- Thunderstorm
- Drought
- Heatwave
- Seasonal Changes

---

# Behaviour Entries

Documents observed behaviours.

Examples:

- Hunting
- Migration
- Nest Building
- Cooperation
- Territorial Defense
- Courtship

---

# Evolution Entries

Tracks important evolutionary milestones.

Examples:

- New Species
- Extinction
- Adaptive Radiation
- Divergence Events

---

# World History

Stores significant simulation events.

Examples:

- First Extinction
- First Predator
- Largest Population
- Longest Living Species
- Climate Catastrophe

---

# Entry Structure

Every Encyclopedia Entry contains:

```text
Entry

├── EntryId
├── Category
├── Title
├── Description
├── Unlock State
├── Discovery Date
├── Related Entries
└── Statistics
```

---

# Unlock State

Possible values:

- Hidden
- Discovered
- Complete

Entries become more complete as additional discoveries are made.

---

# Related Entries

Entries may reference:

- Species
- Traits
- Biomes
- Resources
- Events

Cross-references are generated automatically.

---

# Search

The Encyclopedia supports searching by:

- Name
- Category
- Species
- Trait
- Biome

Search implementation belongs to the UI module.

---

# Statistics

Each entry may expose statistics.

Examples:

- Times Observed
- Worlds Found
- Population Peak
- Extinction Count

---

# Persistence

The Encyclopedia belongs to the Player Profile.

Knowledge persists across all worlds.

---

# Determinism

Entries are unlocked exclusively through deterministic Discovery events.

---

# Serialization

The Encyclopedia must be fully serializable.

Entries are stored independently from World save files.

---

# Design Constraints

The Encyclopedia must remain:

- deterministic
- profile-based
- data-driven
- simulation independent
